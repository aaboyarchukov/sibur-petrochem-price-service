**Как формируется candidate_count**

Формула становится кандидатом, если:

- строка спроса имеет contract = Formula;
- формула не помечена Неактивна = X;
- совпадает клиент;
- совпадает материал напрямую либо через material_groups.xlsx;
- формула уже начала действовать: period ≥ valid_from;
- будущие формулы не рассматриваются;
- просроченные формулы сохраняются как возможный fallback.

**Как выбирается формула в results**

- Успешная актуальная формула.
- material выше group_m.
- Более новая created_at.
- Более новая valid_from.
- При полном равенстве - минимальный formula_id.
- Если актуальной успешной нет - успешная просроченная формула с максимальной valid_to.
- Если ничего не рассчиталось - наиболее приоритетная ошибка с пустой ценой.

- сроки связи группы: datuv ≤ period ≤ datub.

**Расчёт value по типу компонента**

| **Тип**          | **Источник и правило**                                                                                     |
| ---------------- | ---------------------------------------------------------------------------------------------------------- |
| H, A, B, C, D, E | formula_components.xlsx / Значение; пустое значение сейчас принимается равным 0                            |
| 1                | имя из formula_components → код из quote_mapping → значение из quotes.xlsx                                 |
| 5                | значение из currency_rates.xlsx; валюта определяется по полю Валюта, техническому ID либо имени переменной |
| 7                | не рассчитывается: нужен внешний источник Pricing/SAP                                                      |
| другой тип       | ошибка Неподдерживаемый тип компонента                                                                     |

**Выбор котировки или курса**

- Минимальная абсолютная разница с period.
- При одной дате: Факт → ОФ → ППР.
- При равенстве предпочтение прошлой дате перед будущей.
- Для котировок дополнительно выбирается самая поздняя tech_load_ts.

**Важно:** сроки формул не продлеваются. Но сроки компонентов в текущем коде технически продлеваются до конца горизонта спроса через effective_component_valid_to. Это отдельное допущение текущей реализации.

**Статусы**

**results.status**

| **Статус**                       | **Значение**                                                         |
| -------------------------------- | -------------------------------------------------------------------- |
| CALCULATED                       | выбрана успешная актуальная формула                                  |
| CALCULATED_WITH_EXPIRED_FORMULA  | использована успешная просроченная формула                           |
| CALCULATED_WITH_FORMULA_CONFLICT | несколько равноприоритетных формул; выбрана с минимальным formula_id |
| COMPONENT_ERROR                  | не получено значение компонента                                      |
| INVALID_FORMULA                  | выражение не удалось вычислить                                       |
| FORMULA_NOT_FOUND                | кандидат не найден                                                   |
| SPOT_NOT_CALCULATED              | строка SPOT, расчёт не выполнялся                                    |

**selection_reason**

- ACTUAL_SUCCESSFUL_FORMULA
- LATEST_EXPIRED_SUCCESSFUL_FORMULA
- TECHNICAL_TIE_BREAK
- NO_SUCCESSFUL_FORMULA

**formula_candidates.status**

- CALCULATED
- COMPONENT_ERROR
- INVALID_FORMULA

**component_values.status**

- OK
- ERROR
